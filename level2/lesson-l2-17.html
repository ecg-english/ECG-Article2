17. 音の脱落「リダクション（短縮）」②

wanna, gonna, hafta などの口語表現

なぜリダクション（短縮）が重要なのか？

英語では、日常会話で単語が短縮されることが頻繁にあります。これを「リダクション（短縮）」または「Contraction」と言います。これを理解することで、ネイティブの自然な会話を聞き取れるようになります。

リダクション（短縮）が重要な理由

1. ネイティブの自然な会話を聞き取る

リダクション（短縮）を理解することで、ネイティブの自然な会話を聞き取りやすくなります。自分で正しく発音できる短縮形は聞き取れるようになります。

2. 自然な発音

リダクション（短縮）を使うことで、ネイティブのような自然な発音ができます。これが「ネイティブらしく聞こえる」最大の秘密です。

3. 流暢さの向上

リダクション（短縮）を使うことで、発音が滑らかになり、流暢さが向上します。すべての音を発音するよりも、自然で流暢な発音になります。

よくある間違い

カタカナ読み：「ウォント トゥ」（すべての音を発音）

実際の音：[ˈwɑntə] → 「ウォナ」（「want to」が「wanna」に短縮）

修正方法：「want to」を「wanna」と短縮して発音する

🗣️ 話し言葉

  • 短縮形を使う
  • 音が短縮される

リダクション（短縮）とは？

リダクション（短縮）は、日常会話で単語が短縮される現象です。主に、機能語が短縮されることが多いです。正式には「Contraction（コントラクション）」と呼ばれます。

リダクション（短縮）の特徴

1. 日常会話で頻繁に使われる

リダクション（短縮）は、日常会話で頻繁に使われます。フォーマルな場面では使わないこともあります。カジュアルな会話で特に多く使われます。

例：I want to go → 「I wanna go」

日常会話では「want to」を「wanna」と短縮して発音することが多いです。

2. 機能語が短縮される

機能語（want to, going to, have to など）が短縮されることが多いです。内容語は短縮されにくいです。

例：want to → 「wanna」

機能語は意味よりも構造を支えるため、短縮されやすくなります。

3. 書き言葉では使わない

リダクション（短縮）は、話し言葉で使われます。書き言葉では使わないことが多いです。ただし、カジュアルなメールやチャットでは使われることもあります。

フォーマルな文書では、短縮形を使わず、完全な形で書きます。

4. 発音が滑らかになる

リダクション（短縮）を使うことで、発音が滑らかになり、流暢さが向上します。これがネイティブの自然な会話を作り出します。

短縮形を使うことで、音がつながりやすくなり、発音が滑らかになります。

リダクション（短縮）の仕組み

仕組み1：音の結合

2つの単語の音が結合して、1つの音になります。例：「want to」→「wanna」

音が結合することで、短縮形が生まれます

仕組み2：音の脱落

一部の音が脱落して、短縮形になります。例：「going to」→「gonna」（「ing」と「to」の一部が脱落）

音が脱落することで、短縮形が生まれます

仕組み3：音の変化

音が変化して、短縮形になります。例：「have to」→「hafta」（「v」が「f」に変化）

音が変化することで、短縮形が生まれます

リダクション（短縮）の例

リダクション（短縮）の具体例を見てみましょう。日常会話で頻繁に使われる短縮形を確認します。

基本例

1. want to → wanna

書き方：I want to go

カタカナ読み：「アイ ウォント トゥ ゴー」

実際の音：[aɪ ˈwɑntə goʊ] → 「アイ ウォナ ゴー」（「want to」が「wanna」に短縮）

「want to」→「wanna」

「want」と「to」が結合して「wanna」になります。「t」と「t」が結合し、「o」が脱落します。

2. going to → gonna

書き方：I'm going to go

カタカナ読み：「アイム ゴーイング トゥ ゴー」

実際の音：[aɪm ˈgɔnə goʊ] → 「アイム ゴナ ゴー」（「going to」が「gonna」に短縮）

「going to」→「gonna」

「going」の「ing」が脱落し、「to」の「t」が「n」と結合して「gonna」になります。

3. have to → hafta

書き方：I have to go

カタカナ読み：「アイ ハブ トゥ ゴー」

実際の音：[aɪ ˈhæftə goʊ] → 「アイ ハフタ ゴー」（「have to」が「hafta」に短縮）

「have to」→「hafta」

「have」の「v」が「f」に変化し、「to」の「o」が脱落して「hafta」になります。

4. got to → gotta

書き方：I got to go

カタカナ読み：「アイ ゴット トゥ ゴー」

実際の音：[aɪ ˈgɑtə goʊ] → 「アイ ガタ ゴー」（「got to」が「gotta」に短縮）

「got to」→「gotta」

「got」と「to」が結合して「gotta」になります。「t」と「t」が結合し、「o」が脱落します。

その他の例

5. should have → shoulda

書き方：I should have gone

実際の音：[aɪ ˈʃʊdə goʊn] → 「アイ シュダ ゴーン」（「should have」が「shoulda」に短縮）

「should」と「have」が結合して「shoulda」になります。「h」が脱落し、「v」が「a」に変化します。

6. could have → coulda

書き方：I could have done it

実際の音：[aɪ ˈkʊdə dʌn ɪt] → 「アイ クダ ダンイット」（「could have」が「coulda」に短縮）

7. would have → woulda

書き方：I would have said

実際の音：[aɪ ˈwʊdə sed] → 「アイ ウダ セッド」（「would have」が「woulda」に短縮）

「would」と「have」が結合して「woulda」になります。「h」が脱落し、「v」が「a」に変化します。

8. let me → lemme

書き方：Let me see

実際の音：[ˈlemi si] → 「レミー シー」（「let me」が「lemme」に短縮）

「let」と「me」が結合して「lemme」になります。「t」が脱落し、「m」と「m」が結合します。
